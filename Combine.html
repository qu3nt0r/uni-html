Bootstrap 5 Components

A comprehensive guide to essential Bootstrap components with live examples.

Bootstrap 5 Overview

Bootstrap 5 is a free, open-source front-end framework that provides pre-built CSS classes and JavaScript components to help you build responsive, modern websites quickly.

Key Concepts

What It Does

Bootstrap gives you a toolkit of ready-made styles and interactive components so you don't have to write everything from scratch. Think of it as a collection of LEGO blocks for web design.

Responsive Design

Your code uses Bootstrap's grid system (the col-md-4 classes in your cards section). This automatically adjusts your layout based on screen size—your 3-column card layout becomes single-column on phones, maintaining usability across all devices.

Component Library

Your HTML showcases several Bootstrap components:

  • Navbar: Navigation bar with collapsible menu (try resizing your browser)
  • Carousel: Image slider with automatic transitions
  • Modal: Pop-up dialog that appears without page reload
  • Cards: Flexible content containers
  • Forms: Styled input fields and buttons
  • Alerts: Pre-styled notification boxes

Utility Classes

Classes like py-5 (padding vertical), bg-light (light background), and d-block (display block) apply styling instantly—no custom CSS needed.

JavaScript Integration

Bootstrap includes built-in JavaScript for interactive features. Your carousel auto-rotates, the modal opens/closes, and the navbar collapses on mobile—all handled automatically through data-bs-* attributes.

Why Use It?

  • ✅ Saves development time
  • ✅ Ensures consistent, professional appearance
  • ✅ Handles responsive design automatically
  • ✅ Large community support and documentation

Carousel - Image Slider

Component/Section Documentation

Title: Carousel

Purpose: Displays a rotating set of images or content. Useful for showcasing multiple highlights without using extra vertical space.

Behavior & Usage
  • The carousel auto-rotates if data-bs-ride="carousel" is set.
  • Controls and indicators allow manual navigation.
  • Ensure images are similar aspect ratio for consistent layout.
Accessibility
  • Add descriptive alt text to images for screen readers.
  • Consider pausing auto-rotation for long content or provide user controls.
Example markup
<div id="carouselExample" class="carousel slide" data-bs-ride="carousel">
                <div class="carousel-inner">
                    <div class="carousel-item active">
                    <img src="..." class="d-block w-100" alt="Description">
                    </div>
                    <!-- more items -->
                </div>
                <button class="carousel-control-prev" type="button" data-bs-target="#carouselExample" data-bs-slide="prev"><span class="carousel-control-prev-icon"></span></button>
                <button class="carousel-control-next" type="button" data-bs-target="#carouselExample" data-bs-slide="next"><span class="carousel-control-next-icon"></span></button>

Modal - Dialog Box

Component/Section Documentation

Title: Modal

Purpose: Presents a dialog overlay for user actions such as confirmation, forms, or important messages without navigating away from the current page.

Behavior & Usage
  • Open via data-bs-toggle="modal" and data-bs-target="#modalId" or JavaScript APIs.
  • Use appropriate buttons for actions (e.g., Save, Cancel).
Accessibility
  • Give the modal a clear title with .modal-title and ensure focus management (Bootstrap handles most automatically).
  • Include aria-label on close buttons where needed.
Example markup
<button data-bs-toggle="modal" data-bs-target="#exampleModal">Open Modal</button>

                <div class="modal fade" id="exampleModal" tabindex="-1">
                <div class="modal-dialog">
                    <div class="modal-content">
                    <div class="modal-header"><h5 class="modal-title">Title</h5><button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button></div>
                    <div class="modal-body">Content</div>
                    <div class="modal-footer"><button class="btn btn-secondary" data-bs-dismiss="modal">Close</button></div>
                    </div>
                </div>
                </div>

Forms - User Input

Component/Section Documentation

Title: Forms

Purpose: Collect user input with accessible, styled form controls. Bootstrap provides layout utilities and validation styles.

Behavior & Usage
  • Use .form-control, .form-label, and spacing utilities for consistent layout.
  • Leverage client-side validation classes like .is-invalid and .is-valid for feedback.
Accessibility
  • Associate labels with inputs using the for and id attributes.
  • Use descriptive placeholder text and ARIA attributes when needed.
Example markup
<form>
                    <label class="form-label" for="email">Email</label>
                    <input id="email" type="email" class="form-control" placeholder="Enter email">
                    <!-- validation feedback markup -->
                </form>

Alerts - Messages

Component/Section Documentation

Title: Alerts

Purpose: Describe the purpose of this section/component (what it renders and why). State where it is used in the application and any important invariants.

Bootstrap Alerts — Detailed Explanation

What is an alert: Bootstrap "alerts" are a lightweight, flexible way to provide feedback messages to users (success, info, warning, danger, etc.). Alerts use the .alert base class plus a contextual class such as .alert-success, .alert-warning, .alert-danger, etc.

Structure & semantics:

  • Basic structure: <div class="alert alert-* " role="alert">Message</div>
  • The role="alert" attribute ensures assistive technologies announce the message immediately.

Dismissible alerts:

To make an alert dismissible add .alert-dismissible and include a close button with data-bs-dismiss="alert". Optionally include .fade and .show for animation.

Accessibility
  • Include role="alert" for important live messages.
  • Dismiss buttons should have accessible labels (e.g., aria-label="Close").
Behavior notes
  • Alerts are presentational and do not persist across reloads.
  • Use Bootstrap JS APIs for programmatic dismissal if needed.

Examples (code)

<div class="alert alert-success" role="alert">
                    Success! Your action was completed.
                </div>

                <div class="alert alert-warning alert-dismissible fade show" role="alert">
                    Warning: Check the entered information.
                    <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
                </div>

                <div class="alert alert-danger" role="alert">
                    <h4 class="alert-heading">Error</h4>
                    <p>There was a problem processing your request. Please try again later.</p>
                    <hr>
                    <p class="mb-0">If the problem persists, contact support.</p>
                </div>
